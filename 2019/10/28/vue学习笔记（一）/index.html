Vue学习笔记（一）

发表于 2019-10-28 | 分类于 code |

Vue学习笔记

ide选用

之前就听说过vscode的大名，奈何一直平时工作是java，一直用的是idea，没有领略过vscode的风采，这次想都没想就直接上了vscode，下了安装包就惊了，一共才50多M，这跟idea、eclipse动不动就大几百M的安装包相差甚远。然后就是vscode的使用，感觉有点类似sublime text的风格，其次vscode是个开源的ide,这也使它有了良好的社区以及丰富的组建，比如vue就用到了vetur、ESLint，总的来说，vscode真香！！！看看后工作中写java用vscode是否也方便。

Vue项目创建

安装内容 ：

  • node.js 主要是为了使用后面的npm和webpack，跟vue其实关系不大
  • npm 包管理工具，类似java中的maven
  • webpack 打包工具，类似maven中的打包以及包管理
  • vue 可以不用安装，直接引用也可以

Vue中的常用指令

动态绑定 v-bind
<div id="app-2">
  <span v-bind:title="message">
    鼠标悬停几秒钟查看此处动态绑定的提示信息！
  </span>
</div>

var app2 = new Vue({
  el: '#app-2',
  data: {
    message: '页面加载于 ' + new Date().toLocaleString()
  }
})
条件 v-if
<div id="app-3">
      <p v-if="seen">现在你看到我了</p>
</div>    

var app3 = new Vue({
  el: '#app-3',
  data: {
    seen: true
  }
})
循环 v-for

类似于java中的foreach，使用 

app4.todos.push{{text : 'pushtest'}}

还可以动态插入数据

<div id="app-4">
  <ol>
    <li v-for="todo in todos">
      `{{ todo.text }}`
    </li>
  </ol>
</div>

var app4 = new Vue({
  el: '#app-4',
  data: {
    todos: [
      { text: '学习 JavaScript' },
      { text: '学习 Vue' },
      { text: '整个牛项目' }
    ]
  }
})
用户输入处理 v-on
<div id="app-5">
      <p>{{ message }}</p>
      <button v-on:click="reverseMessage">反转消息</button>
</div>

var app5 = new Vue({
  el: '#app-5',
  data: {
    message: 'Hello Vue.js!'
  },
  methods: {
    reverseMessage: function () {
      this.message = this.message.split('').reverse().join('')
    }
  }
})
输入展示双向绑定 v-model
div id="app-6">
      <p>`{{ message }}`</p>
      <input v-model="message">
</div>

var app6 = new Vue({
  el: '#app-6',
  data: {
    message: 'Hello Vue!'
  }
})
自定义组件 Vue.component
<div id="app-7">
  <ol>
    <todo-item
      v-for="item in groceryList"
      v-bind:todo="item"
      v-bind:key="item.id"
    ></todo-item>
  </ol>
</div>

Vue.component('todo-item', {
  props: ['todo'],
  template: '<li>`{{ todo.text }}`</li>'
})

var app7 = new Vue({
  el: '#app-7',
  data: {
    groceryList: [
      { id: 0, text: '蔬菜' },
      { id: 1, text: '奶酪' },
      { id: 2, text: '随便其它什么人吃的东西' }
    ]
  }
})

学习总结

主要了解vue的项目构建以及一些主要的语法糖，code的思维其实大同小异，无论是前端还是后端，无论是编译型语言还是脚本型语言，现在都趋于相同思维。相比于很久以前用于写前端的jquery那种东西，不得不说所有语言低层次的包，框架都在为开发者聚焦业务做贡献，所需要关注的底层越来越少，编码方式越来越简单，这也不得不归功于很多开源大佬以及社区做出的贡献。最后说一句，解释型语言的部署是真的快！！！ctrl + s一秒部署完毕，后端即使java支持热部署，还是跟解释型语言没得比！！
